# [PRD] EverLaw Edu: AI 에이전트 기반 자율 갱신형 컴플라이언스 솔루션

## 1. 제품 개요 (Product Overview)
*   **제품명**: EverLaw Edu (에버로 에듀)
*   **슬로건**: "법은 변해도, 교육은 멈추지 않도록 — 실시간 법령 대응 에이전트"
*   **핵심 가치**: 
    1. 규제 변화에 대한 **즉각적 대응(Zero-latency)**
    2. 교육 콘텐츠 유지보수 **비용 절감(Cost Efficiency)**
    3. 법적 리스크 **사전 차단(Risk Management)**

---

## 2. 비즈니스 배경 및 목표 (Business Goals)
*   **배경**: 기업의 법정 의무 교육은 매년 개정되는 법령에 따라 콘텐츠를 수정해야 함. 기존 방식은 [**전문 인력의 수동 검토 -> 콘텐츠 재제작 -> 재배포**] 과정을 거치며 평균 2~4주의 시차가 발생하고 높은 비용이 소요됨.
*   **목표**:
    1. 법령 개정 후 **24시간 이내** 교육 콘텐츠 자동 업데이트율 95% 달성.
    2. 수동 운영 공수 대비 **70% 이상의 리소스 절감**.
    3. AI 생성 콘텐츠의 관리자 최종 승인율 **90% 이상** 유지.

---

## 3. 사용자 정의 (User Personas)
1.  **기업 교육 담당자 (Admin)**: 법령 변화를 일일이 확인하기 힘든 인사/노무 담당자. AI가 제안한 수정안을 최종 검토 및 승인함.
2.  **임직원 (Learner)**: 본인 직무와 관련된 최신 법규를 가장 빠르고 정확하게 학습해야 하는 사용자.

---

## 4. 핵심 기능 요구사항 (Functional Requirements)

### 4.1 에이전트 워크플로우 (Agentic Workflow)
*   **[F-1] 실시간 법령 스캐너**: 국가법령정보센터 API 및 주요 정부 부처 RSS를 상시 모니터링하여 개정안 감지.
*   **[F-2] 지능형 차분 분석(Impact Analysis)**: 개정된 법 조항이 기존 교육 커리큘럼의 어느 부분(텍스트, 이미지 설명, 퀴즈)에 영향을 주는지 분석.
*   **[F-3] 자율 콘텐츠 생성**: RAG(Retrieval-Augmented Generation) 기술을 활용하여 개정 법령 기반의 신규 강의안 및 퀴즈 생성.
*   **[F-4] 자가 검증 시스템**: 생성된 콘텐츠의 법적 근거가 정확한지 AI가 역으로 교차 검증 수행.

### 4.2 사용자 경험 (User Experience)
*   **승인 워크플로우**: 관리자가 변경 전/후를 비교(Side-by-Side)하고 한 번의 클릭으로 전체 학습자에게 배포하는 UI.
*   **개인화 푸시 알림**: 학습자의 직무와 연관된 법령이 바뀔 때만 Flutter 기반의 맞춤형 알림 전송.

---

## 5. 기술 사양 (Technical Specifications)

### 5.1 Back-end (Spring Boot)
*   **Architecture**: MSA 지향 구조로 법령 수집 서비스와 콘텐츠 생성 서비스를 분리.
*   **Data Integrity**: JPA를 활용한 교육 이력 관리 및 콘텐츠 버전 관리(Versioning) 시스템 구축.
*   **AI Integration**: FastAPI 기반 LLM 서버와 gRPC 또는 REST API를 통한 고성능 통신. LangChain을 활용한 RAG 파이프라인 및 에이전트 워크플로우 구축.

### 5.2 Front-end (Flutter)
*   **Single Codebase**: Android, iOS, Web 동시 대응으로 기업 내 멀티 디바이스 환경 지원.
*   **Dynamic UI**: AI가 생성한 마크다운(Markdown) 기반 학습 자료를 실시간으로 렌더링하는 컴포넌트 구현.

---

## 6. 리스크 관리 및 대응 (Risks & Mitigation)
*   **Hallucination (환각 현상)**: AI가 법령을 잘못 해석할 가능성. 
    *   *대응*: **Human-in-the-loop** 프로세스를 강제하여 관리자 승인 없이는 배포가 불가능하도록 설계.
*   **Source API Failure**: 공공 API 장애 상황. 
    *   *대응*: 주요 법률 포털 크롤링 엔진을 백업으로 운영하여 데이터 수집의 연속성 확보.

---

## 7. 로드맵 (Roadmap)
*   **Phase 1 (MVP)**: 산업안전보건법 등 핵심 법정 의무 교육 대상 자동 업데이트 구현.
*   **Phase 2**: 기업별 사규 및 내부 규정 업데이트 지원 기능 확장.
*   **Phase 3**: 다국어 지원을 통한 글로벌 컴플라이언스 시장 진출.
