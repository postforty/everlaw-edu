# [PRD] EverLaw Edu: 최신 법령 DB 기반 교육 콘텐츠 자율 생산 및 컴플라이언스 솔루션

## 1. 제품 개요 (Product Overview)
*   **제품명**: EverLaw Edu (에버로 에듀)
*   **슬로건**: "가장 신선한 법령으로, 가장 정확한 교육을 — 실시간 법령 기반 콘텐츠 생산 공장"
*   **핵심 가치**: 
    1. **절대적 팩트 신뢰도(Absolute Truth)**: 실시간 갱신되는 국가 표준 법령 DB를 지식의 원천(Source of Truth)으로 삼아 교육 팩트 오류를 원천 차단.
    2. **자율적 콘텐츠 대량 생산(Generative Scalability)**: 법령 전문에서 가상 현장 시나리오 기반의 모의 평가 퀴즈를 무(無)에서 유(有)로 동적 자동 창작.
    3. **비즈니스 콜드 스타트 제거(SaaS Ready)**: 기존 문제 은행이 없는 기업 고객도 최신 법령 DB를 통해 컴플라이언스 퀴즈 자료 즉시 생산 배포 가능.

---

## 2. 비즈니스 배경 및 목표 (Business Goals)
*   **배경**: 기존 의무 교육 콘텐츠의 수동 교정 방식은 기업이 기존 교안을 보유하고 있어야 한다는 한계가 존재하며, 파편화된 문서 포맷으로 인해 파싱 노이즈가 컸음. 이에 국가 표준 법령 전문을 벡터 DB에 항시 최신으로 유지하고, 이를 근거로 교육 콘텐츠를 규격화하여 자율적으로 "생산"하는 스마트 팩토리 모델로 도약함.
*   **목표**:
    1. 법령 개정 고시 즉시 **벡터 DB 실시간 SQL-level Upsert 100%** 달성.
    2. 최신 법령 근거 **학습용 4지선다 퀴즈의 원클릭 자동 생산율 98%** 달성.
    3. AI 자가 팩트체크 검증 노드를 통한 법령 왜곡 차단 및 **환각율 0.0% 지향**.

---

## 3. 사용자 정의 (User Personas)
1.  **기업 교육 담당자 (Admin)**: 사내 컴플라이언스 교육을 기획하는 담당자. 최신 법령 DB를 바탕으로 원하는 주제의 맞춤형 퀴즈를 자율 생산하고 최종 배포를 승인함.
2.  **임직원 (Learner)**: 갓 개정된 최신 법령 지식이 100% 반영된 흥미로운 시나리오 기반 퀴즈를 풀며 학습하는 사용자.

## 4. 핵심 기능 요구사항 (Functional Requirements)

### 4.1 에이전트 워크플로우 (Agentic Workflow)
*   **[F-1] 법령 DB 스캐너 및 Upsert (실시간 & Daily 스케줄링)**: 국가법령정보센터 및 GitHub 커밋 감지 즉시 기동되는 실시간 파이프라인 외에, **"매일 지정된 시점에 변경 내역 유무를 정기 점검하는 일배치(Daily Batch) 스케줄링 기능"**을 병행 제공합니다. 수정된 파일의 **전문(Whole File)을 다운로드**하고 **동일한 계층형 5단계 마크다운 파이프라인**을 통해 **'조(Article)' 단위로 정밀 청킹**합니다. 이후 **"스마트 SHA-256 해시 CDC(Change Data Capture) 스킵"** 로직을 거쳐 내용이 변경된 청크에 대해서만 고유 비즈니스 키(`law_{law_name}_{article}`)를 기준으로 벡터 DB에 **물리적 SQL-level 선삭제(DELETE) 후 삽입(INSERT)**함으로써 멱등성을 100% 보존합니다.
*   **[F-2] 지능형 법령-콘텐츠 매핑(Impact Mapping)**: 새로 감지되거나 변경된 법령 조항이 사내 교육 카테고리(예: 안전보건, 근로기준 등)와 어떻게 매핑되는지 영향 관계 분석.
*   **[F-3] 법령 기반 자율 콘텐츠 생산 (Ground Truth Generation - 듀얼 모드 기동)**: 최신 법령 벡터 DB를 근거 지식으로 삼아, 스토리텔링과 현장 시나리오가 가미된 **"학습자용 4지선다형 모의 퀴즈"**를 무(無)에서 유(有)로 자동 창작 생산합니다. 이때 생성 엔진은 **정적 전체 문제 은행 구축**과 **동적 실시간 갱신**을 통합 지원하는 **듀얼 모드(Dual-Mode)**로 가동됩니다.
    *   **듀얼 모드 생성 아키텍처**:
        1. **[모드 A] 벌크 종합 문제 은행 빌더**: 새로운 법령이 시스템에 최초 시딩되는 시점에, 전체 법 조문 조항을 RAG로 자동 순회하며 모든 필수 의무 사항에 대해 1:1 또는 1:N으로 매핑되는 4지선다형 퀴즈를 대량 생산하여 통합 퀴즈 DB(`quiz_bank`)에 사전 적재합니다.
        2. **[모드 B] 실시간 델타 핫스왑 업데이트**: 법령 스캐너가 실시간 변경 또는 개정을 감지할 때, 갱신된 델타 조항을 지식 근거로 삼아 기존 퀴즈 DB(`quiz_bank`)에서 해당 조항을 가리키는 기존 퀴즈 레코드를 정밀 추적(Targeting)하여 퀴즈 지문의 규제 수치, 정답 인덱스, 해설을 AI가 자동으로 리라이팅 및 최신화(핫스왑)합니다.
    *   **4지선다 퀴즈 세부 생성 사양**:
        1. 딱딱한 법령 조문을 가상의 현장 사례 및 인물(예: 김 대리)의 사건을 다룬 **스토리텔링형 지문**으로 재구성합니다.
        2. 질문과 4개의 객관식 보기 옵션(정답 1개, 매력적인 오답 3개)을 제공합니다.
        3. 사용자가 생각을 유도하도록 돕는 **"힌트(Hint)"**를 제공합니다.
        4. 법령의 핵심 의미와 근거 조항 번호가 명확히 명시된 **"상세 해설(Explanation)"**을 정교하게 포함하여 퀴즈 패키지 형태로 자동 생산합니다.
*   **[F-4] AI 자가 검증 시스템 (Auto-Validation)**: 생산된 교육용 퀴즈의 수치, 의무 의도가 원본 법령 팩트와 1대1 일치하는지 AI 가 검증 감사하여 환각 지수(Hallucination Score) 산출.

### 4.2 사용자 경험 (User Experience)
*   **[U-1] 근거-퀴즈 Side-by-Side 승인 UI (관리자용)**: 화면 좌측에는 **최신 법령 팩트 데이터**, 우측에는 **AI가 생산한 신규 퀴즈**를 배치하여, 관리자가 법적 근거와 퀴즈의 일치 여부를 직관적으로 검증하며 즉시 승인 및 배포할 수 있는 UI.
*   **[U-2] 사용자 직무별 푸시 알림**: 학습자 직무와 매핑된 최신 개정 법령 및 신규 생산된 퀴즈에 대한 모바일 알림 전송.
*   **[U-3] 학습자용 스토리텔링 4지선다 퀴즈 UI**:
    *   학습자가 4지선다 항목 중 하나를 클릭하면 **즉각적인 정답/오답 비주얼 피드백(색상 변경 및 사운드/진동 등)**을 제공합니다.
    *   **정답을 맞춘 경우**: 정답 연출 애니메이션과 함께 **약 0.8초~1.0초의 모션 딜레이**를 부여한 후, 다음 퀴즈 카드를 화면에 매끄럽게 페이드인 렌더링합니다.
*   **[U-4] 오답 복습 해설 및 하단 컨텍스트 바인딩 AI 채팅창**:
    *   **오답을 선택한 경우**: 선택한 보기 아래에 **"상세 해설(Explanation)"** 패널이 부드러운 애니메이션(Slide-in)과 함께 활성화됩니다.
    *   **AI 코칭 챗봇 배치**: 해설 영역 바로 아래에는 **AI 도우미와 대화할 수 있는 인라인 채팅창**이 동적으로 배치됩니다.
    *   **컨텍스트 연동**: 학습자가 채팅창에 궁금한 내용을 질문하면, 해당 문제의 `[스토리텔링 내용]`, `[관련 법령 조항]`, `[사용자가 선택한 오답 사유]`가 챗봇 컨텍스트에 자동 바인딩되어 오답 원인 분석 및 법령 해석을 1:1 맞춤형 과외처럼 피드백합니다.
*   **[U-5] 상단 오답노트 복습 네비게이션**:
    *   학습 화면 상단 헤더(AppBar 영역)에 직관적인 디자인의 **"오답노트"** 버튼을 상시 배치합니다.
    *   해당 버튼을 탭하면 로컬 디바이스 저장소에 누적 저장된 오답 데이터를 기반으로 틀렸던 문제를 유형별/법령별로 정렬하여 다시 풀어보거나(재도전), 이전 대화 기록을 모아 복습할 수 있는 복습 전용 화면으로 매끄럽게 라우팅됩니다.
*   **[U-7] 오답노트 개별 문제 삭제 및 아카이빙 (Learner Deletion & Mastery Archiving)**:
    *   학습자는 오답노트 목록에서 학습을 완수하여 완전히 숙지한 문제에 대해 **"삭제/완료"** 버튼을 눌러 개별 오답 항목을 오답노트에서 직접 지우거나 아카이빙할 수 있습니다.
    *   사용자가 삭제를 수행하면 로컬 저장소에서 해당 오답 기록이 즉시 제거(또는 완료 마킹)되어 화면에서 스냅하게 페이드아웃되며, 서버 DB에도 비동기적으로 싱크 처리됩니다. 학습 마스터(Mastery) 처리에 따른 삭제 시, 관련 법령 조항의 개인 취약 지수(`weakness_score`)를 점진적으로 안전 구역으로 보정 갱신합니다.
*   **[U-8] 연속 정답 기반 지능형 오답노트 자동 졸업 (Adaptive Mastery Graduation System)**:
    *   학습자가 수동으로 오답을 삭제하지 않더라도, 특정 취약 법령 조항에 대해 **유사도가 높은 변형 문제(`[U-6]`)를 연속 정답 기준(예: 연속 3회 정답)으로 맞출 경우, 해당 조항의 모든 오답 문제를 오답노트 리스트에서 자동으로 아카이빙(숨김) 처리**하는 지능형 학습 완수 메커니즘을 제공합니다.
    *   시스템이 학습자의 개념 숙지를 자동 진단하며, 조건 충족 시 임직원 앱 화면에 **"개념 정복(Mastery) 성공 및 자동 졸업"** 축하 연출과 함께 개인 취약 지수(`weakness_score`)가 최적의 안전 상태로 자동 청산됩니다.
*   **[U-6] 지능형 개인 맞춤형 변형 퀴즈 (Adaptive AI Quiz Clinic)**:
    *   학습자의 오답 통계를 분석하여 가장 취약한 법령 조항(오답 발생 빈도 최고 조항)을 도출합니다.
    *   학습자가 취약점 극복 훈련을 클릭하면, AI가 사용자가 이전에 풀었던 퀴즈의 지문 및 작업 상황과 겹치지 않는 **완전히 새로운 실무 현장 시나리오의 변형 4지선다형 퀴즈를 온더플라이(On-the-fly)로 즉석 생성**해 줍니다. 
    *   이를 통해 학습자는 단순 정답 번호 암기를 탈피하고 법령 규격의 본질을 완벽하게 극복했는지 입체적으로 훈련하며, 통과 시 개인별 취약 조항 지수가 안전 구역으로 차감 갱신됩니다.

## 5. 기술 사양 (Technical Specifications)
*   **Back-end**: Spring Boot 3.x (JPA, PostgreSQL) & FastAPI (AI Engine)
*   **AI Stack**: Google Gemini 3.1 Flash-Lite, Ollama (bge-m3 Embedding), LangGraph 워크플로우
*   **Storage (서버 데이터베이스 & 클라이언트 로컬 디바이스 저장소)**:
    *   **Server Database**: PostgreSQL (pgvector)에 **법령 전문 컬렉션(`law_documents`)**, 그리고 전체 조항별로 생성된 대량의 4지선다 문제를 통합 영속화하고 이력을 보존하는 **통합 문제 은행 컬렉션(`quiz_bank`)**을 구축 및 통합 운영합니다.
    *   **Client Database (Local Storage)**: 학습자의 오프라인 지원, 실시간 응답성 보장 및 퀴즈 결과 영속성을 위해 **로컬 디바이스 저장소(SQLite, Hive 또는 편의상 SharedPreferences 등)**를 탑재합니다.
*   **오답 데이터 로컬 스키마 (Client Schema)**:
    *   틀린 문제는 즉시 사용자의 로컬 디바이스 저장소에 다음과 같은 규격의 데이터 구조로 직렬화되어 보관됩니다.
    ```json
    {
      "id": "String (UUID, 로컬 기록 고유 식별자)",
      "quizId": "String (UUID, 서버 측 원본 퀴즈 식별자)",
      "question": "String (스토리텔링 질문 본문)",
      "options": "List<String> (4지선다 보기 항목 배열)",
      "answerIndex": "Int (0~3 사이의 정답 인덱스)",
      "selectedIndex": "Int (학습자가 선택한 오답 인덱스)",
      "hint": "String (생성된 힌트 문구)",
      "explanation": "String (상세 법적 근거 해설)",
      "lawReference": "String (근거 법령 이름 및 관련 조항 정보)",
      "incorrectAt": "String (ISO-8601 타임스탬프, 틀린 시점)",
      "isSynced": "Boolean (서버 동기화 완료 여부 플래그)",
      "isArchived": "Boolean (자동 아카이빙/숨김 처리 여부)"
    }
    ```

## 6. 리스크 관리 및 대응 (Risks & Mitigation)
*   **LLM 출력의 건조함 (Dryness) 및 퀴즈 편차**: 
    *   *대응*: 페르소나 및 스토리텔링 기법이 탑재된 프롬프트 엔지니어링을 설계하여 딱딱한 법령 문장을 친숙한 현장 예시와 수칙으로 각색합니다. 특히 퀴즈 생성 시 반드시 **명확한 인과관계가 드러나는 4지선다형 스토리텔링 지문**과 **자연스러운 힌트/해설**을 설계하도록 프롬프트 퓨샷(Few-shot) 가이드라인을 강제하여 일관된 고품질 콘텐츠 품질을 유지합니다.
*   **확장된 환각 위험**:
    *   *대응*: 자가 검증 노드에서 법 조문 원본 팩트 대비 왜곡 지수를 0.0%로 엄격 통제하며, 법령 수치(숫자, 기한)가 1%라도 어긋날 시 강제 반려 및 관리자 경고등(🔴 Red Flag) 점등.
*   **로컬 오답 데이터 유실 및 동기화 지연**:
    *   *대응*: 앱 비정상 종료 시에도 로컬 디바이스 저장소의 트랜잭션 안전성(ACID)을 확보하기 위해 로컬 쓰기 로직에 예외 처리를 철저히 설계하고, 주기적으로 오답 발생 통계를 서버로 안전하게 미러링 동기화하는 백업 프로세스를 운영합니다.
