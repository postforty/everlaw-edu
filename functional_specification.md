# [기능 명세서] EverLaw Edu: 최신 법령 DB 기반 교육 콘텐츠 자율 생산 솔루션

## 1. 문서 개요 (Document Overview)
본 기능 명세서는 개편된 제품요구사항정의서(PRD)의 "최신 법령 DB 기반 콘텐츠 자율 생산 및 컴플라이언스 관리" 비전을 실현하기 위해, 세부 기능 사양과 인공지능 에이전트 시스템(LangGraph)의 동작 메커니즘을 상세히 명세한 백서입니다.

---

## 2. 시스템 아키텍처 개요 (System Architecture Overview)
EverLaw Edu는 국가 표준 법령 DB를 벡터화하여 항상 신선하게 유지하고, 이를 지식 소스로 삼아 실무용 교육 콘텐츠(강의안, 시나리오, 퀴즈)를 자동 창작해내는 동적 생성형 시스템입니다.

1. **Spring Boot (Back-end)**: 비즈니스 도메인 관리, 사용자와 생성 콘텐츠 버전(Snapshot) 매핑, 관리자 승인 상태 처리 및 Flutter 제공 API 서빙.
2. **Python 기반 AI 에이전트 (AI Engine)**: **FastAPI**와 **LangGraph** 기반의 자율 에이전트 파이프라인.
    - **지식 소스(Source of Truth)**: pgvector `law_documents` 컬렉션 (산업안전보건법 전문 등 수록).
    - **생산 결과물**: pgvector `curriculum_documents` 컬렉션 (생산된 교안/퀴즈 수록).
    - **AI Core**: Google Gemini 3.1 Flash-Lite & bge-m3 Embedding
3. **Redis (Cache & Message Broker)**: 동일 법령에 대한 콘텐츠 중복 생산 방지 캐싱, 비동기 큐 처리 및 FCM 푸시 대기 로직.
4. **PostgreSQL (pgvector)**: 법령 전문 및 생성 교안을 저장하는 물리 테이블과 벡터 공간 통합 운영.

## 3. 핵심 기능 명세 (Core Functional Specifications)

### 3.1 AI 에이전트 워크플로우 시스템 (AI Agent Workflow System)
최신 법령 DB를 신선하게 유지하고, 이를 바탕으로 고품질 콘텐츠를 창작 및 교차 검증하는 핵심 파이프라인입니다.

*   **[F-1] 실시간 법령 DB 스캐너 및 SQL-level Upsert**
    *   **기능 요약**: 국가 법령 및 GitHub 커밋을 감지하여 실질 조문을 추출하고, pgvector 테이블에 **물리적 SQL-level 선삭제 후 삽입(Upsert)** 처리를 거쳐 DB 신선도 100% 동기화 유지.
    *   **상세 요건**:
        *   **'조(Article)' 단위 완결 청킹 전략**: 법령 데이터 분할(Chunking) 시 자잘한 단어/문장 단위의 2차 물리 캐릭터 청킹을 철저히 금지하고, 의미론적으로 완전무결한 '조(Article)' 단위(`##### 제N조`)로 한 조각씩 청크를 생성하여 pgvector에 적재. 이를 통해 RAG 검색 후 생성 단계에서의 맥락 단절 및 AI 환각을 근본적으로 예방함.
        *   데이터 삽입 전 동일한 고유 식별자 키(`law_id`, `commit_sha`)의 낡은 행을 강제 DELETE한 후 인서트하여 데이터 중복 축적 원천 차단.
        *   법령 조문 텍스트와 개정 날짜 등의 메타데이터를 유기적으로 바인딩하여 임베딩 적재.

*   **[F-2] 지능형 법령-콘텐츠 영향도 매핑 엔진 (Impact Mapping Engine)**
    *   **기능 요약**: 새로 유입되거나 변경된 법령 조항이 회사의 사내 교육 카테고리와 매핑되는 영향도 수준(High/Medium/Low) 분석.
    *   **상세 요건**:
        *   신규 개정 법령 텍스트를 기준으로 RAG 벡터 역검색을 기동하여, 영향 가능성이 큰 교육 분야나 커리큘럼 카테고리를 핀포인트 매핑.

*   **[F-3] 법령 기반 자율 콘텐츠 생산기 (Law-based Content Generator)**
    *   **기능 요약**: `law_documents` 컬렉션의 가장 신선한 법령 조문을 주입받아, 학습 매력도가 높은 마크다운 교안 및 평가 퀴즈를 무(無)에서 유(유)로 자율 생산.
    *   **상세 요건**:
        *   **스토리텔링 및 시나리오 가미 프롬프트**: 딱딱한 조문 텍스트를 친근한 가상 사고 시나리오와 행동 요령 수칙으로 가공 렌더링.
        *   동일한 법령 기준 중복 생성 방지를 위한 **Redis 캐싱** 적용.
        *   객관식 평가 모의 퀴즈와 정답 및 상세 해설을 동적 구조 데이터(JSON)로 자동 배출.

*   **[F-4] AI 자가 검증 시스템 (Auto-Validation System)**
    *   **기능 요약**: 생산된 콘텐츠가 원본 법령의 팩트(조항 번호, 규제 수치 등)를 훼손했는지 역으로 교차 사실 확인 검증.
    *   **상세 요건**:
        *   수치 비교 감사 체인을 통해 원본 법령 데이터와 1%라도 수치가 어긋나면 환각 점수(Hallucination Score)를 과금 부과하고 반려(🔴 Red Flag) 처리.
        *   감사 소견(Validation Details)을 마크다운 리포트 최하단에 강제 결합 제공.

### 3.2 관리자 시스템 (Admin System)
인사/노무 담당자가 생산된 콘텐츠를 검토하고 안전하게 릴리스하는 통제 센터입니다.

*   **[A-1] 근거 법령-교안 대조 및 승인 워크플로우 (Side-by-Side Reference & Content Review)**
    *   **기능 요약**: RAG의 원천이 된 법령 조문 팩트와 AI가 생산해낸 마크다운 결과물을 좌우로 나란히 대조하여 직관적으로 확인하고 배포 승인.
    *   **상세 요건**:
        *   **좌측 (Left Panel)**: RAG 검색의 근거가 된 '최신 법정 조문 원본 팩트'.
        *   **우측 (Right Panel)**: AI가 생산해낸 '교육용 마크다운 본문 및 모의 퀴즈'.
        *   AI 자가 검증 보고서 접합 제공 및 승인 시 즉각 버전 스냅샷 저장 및 학습자 배포 트리거.

*   **[A-2] 교육 이력 및 스냅샷 버전 관리 (Snapshot Versioning & History)**
    *   **기능 요약**: 교안이 업데이트되더라도 특정 시점에 학습자가 이수한 팩트를 증명하기 위해 과거 버전 스냅샷 영구 보존.
    *   **상세 요건**:
        *   JPA Auditing 및 스냅샷 이력 테이블을 통합하여 법적 감사(Audit) 시 증빙 서류 즉각 출력 가능성 확보.

### 3.3 사용자 시스템 (Learner System)

*   **[U-1] 동적 마크다운 UI 렌더러 (Dynamic UI Renderer)**
    *   AI가 자율 생산한 고품질 스토리텔링 마크다운을 Flutter 모바일 및 웹 화면에 구조화하여 렌더링.
*   **[U-2] 직무별 맞춤형 푸시 알림 (Personalized Notification)**
    *   학습자의 직무 카테고리와 매핑된 신규 생산 교안 및 개정 법령이 발생하면 즉시 FCM 푸시 발송.

## 4. 비기능 요구사항 (Non-Functional Requirements)
1. **정밀성 및 안전성 (Precision & Safety)**:
    *   **Zero-Hallucination & Goldilocks Chunking**: 법령의 숫자, 기한, 의무 규정에 대해 LLM이 허위로 작성하는 행위를 차단하기 위해 팩트 검증 통과(환각 지수 0.0%) 시에만 배포용 락(Lock)이 해제됨. 이를 기술적으로 보장하기 위해 데이터 청킹 시 2차 물리 분할을 차단하고 의미론적 완결 단위인 **'조(Article)' 단위(300자~1,500자 골디락스 존)로 완벽 격리 보존**함.
2. **성능 및 신선도 (Performance & Freshness)**:
    *   원격 법령 개정안 감지 즉시 1분 이내 벡터 DB 동기화 완료 및 5분 이내 신규 교안 자율 생산 완수.
3. **보안 및 규정 준수 (Security)**:
    *   개인정보 규정 및 학습 기록 유출을 방지하기 위한 JWT 인가 및 Mac Mini 로컬 Docker 격리 네트워크 설계.
